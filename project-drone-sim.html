Autonomous UAV simulation & control

A physics-based quadrotor simulation with cascaded control architecture, trajectory tracking, and UAV-inspired design.

Overview

This project focuses on building a high-fidelity quadrotor (UAV) simulation from first principles, with an emphasis on control system design and realistic flight behavior. The system models full 6-DOF rigid body dynamics using quaternions for attitude representation and simulates the response of a drone to control inputs in real time.

The primary objective is to replicate the core architecture of real autonomous UAV systems, including cascaded control loops, trajectory tracking, and modular software structure. Rather than treating the drone as a black box, the project builds each component—from dynamics to control—explicitly to develop a deeper understanding of UAV behavior.

My work on the project

Dynamics modeling

Developed a full nonlinear quadrotor dynamics model in Python, including translational and rotational motion. Attitude is represented using quaternions to avoid singularities, with conversion to Euler angles for control.

Implemented a custom numerical integrator to propagate the system state over time, enabling stable simulation of vehicle motion under varying control inputs.

Cascaded control system design

Designed and implemented a cascaded UAV control architecture, separating position and attitude control:

  • Outer loop (position control): computes desired accelerations based on position and velocity error.
  • Inner loop (attitude control): tracks desired roll, pitch, and yaw using PID controllers to generate body torques.

Position commands are mapped to desired orientation using small-angle approximations:

ax → θdes (pitch)

ay → φdes (roll)

This mirrors the control strategy used in real quadrotors.

PID control & tuning

Implemented reusable PID controller classes and applied them across all control axes (x, y, z, roll, pitch, yaw).

Tuned gains to achieve:

  • fast stabilization from large initial disturbances
  • minimal oscillation in angular rates
  • stable hover and position tracking

Debugged instability issues such as:

  • large initial acceleration spikes
  • oscillatory roll/pitch behavior
  • improper coupling between translational and rotational dynamics

Trajectory tracking & control integration

Extended the controller from simple setpoint stabilization to full position-based navigation, enabling the drone to move to arbitrary (x, y, z) coordinates.

Implemented:

  • velocity feedback in position control loops
  • conversion from desired acceleration to attitude commands
  • thrust control for altitude stabilization

Structured the system to support time-based trajectory inputs (position, velocity, acceleration), aligning with real UAV trajectory tracking methods.

Software architecture

Refactored the project into a modular structure inspired by real UAV flight stacks: 

drone-sim/
├── drone_main.py        # simulation loop, logging, plotting
├── dynamics_fun.py      # 6-DOF rigid body dynamics (quaternion)
├── integrator_fun.py    # numerical state integrator
├── controllers/
│   ├── innerLoop.py     # cascaded position + attitude controller
│   └── PID.py           # reusable PID class
└── transform/           # helper functions (quat <-> Euler, body-to-global, mixer, etc.)

Separated control logic from simulation by:

  • encapsulating the inner-loop controller in its own module
  • passing persistent PID controller objects across time steps
  • structuring inputs as (state, reference, parameters)

This improves scalability and mirrors real systems such as PX4 Autopilot and ArduPilot.

AI-assisted development workflow

Used AI tools (primarily Claude via terminal) to accelerate development and debugging, including:

  • rapid iteration on control architectures
  • diagnosing instability and tuning issues
  • restructuring code into modular components
  • validating control logic against expected UAV behavior

AI was used as an engineering assistant, not a replacement for understanding—each component was derived, implemented, and validated independently.

Integration & testing

Validated controller performance through simulation by:

  • initializing the drone at large attitude disturbances (e.g., 45° roll/pitch)
  • verifying rapid stabilization to zero attitude
  • testing position convergence to arbitrary reference points
  • comparing desired vs. actual accelerations to evaluate controller behavior

Used data visualization (Matplotlib) to analyze:

  • position, velocity, and attitude over time
  • control inputs and system response
  • tracking performance and transient behavior

Results & lessons learned

  • Cascaded control significantly improves stability and modularity compared to single-loop designs.
  • Mapping acceleration to attitude is critical for realistic quadrotor behavior.
  • Controller tuning must account for coupling between translational and rotational dynamics.
  • Numerical artifacts (e.g., acceleration spikes) can arise from initialization and must be carefully interpreted.
  • Clean software architecture is essential as system complexity grows.

Next steps
