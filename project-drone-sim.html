Autonomous UAV simulation & control

Overview

This project develops a high-fidelity quadrotor (UAV) simulation from first principles, with a focus on control system design, trajectory generation, and realistic autonomous flight behavior. The system models full 6-DOF rigid body dynamics using quaternion-based attitude representation and simulates real-time vehicle response to control inputs.

The project evolves beyond simple setpoint stabilization into a structured autonomous control stack, incorporating trajectory-based waypoint navigation and a cascaded control architecture. Instead of commanding discrete position targets, the system generates smooth, time-parameterized trajectories (position, velocity, acceleration), enabling physically consistent and dynamically feasible motion.

The goal is to mirror the architecture of real UAV flight systems by explicitly building each layer—from dynamics to trajectory planning—while maintaining a modular and extensible software design.

My work on the project

Dynamics modeling

  • Developed a full nonlinear quadrotor dynamics model in Python, including coupled translational and rotational motion.
  • Implemented quaternion-based attitude representation to avoid singularities, with conversion to Euler angles for control interpretation.
  • Built a custom numerical integrator to propagate system state, enabling stable simulation under aggressive maneuvers and disturbances.

Cascaded control system design

Designed and implemented a multi-layer UAV control architecture:

  • Outer loop (position control): computes desired accelerations from trajectory tracking error.
  • Inner loop (attitude control): tracks desired roll, pitch, and yaw using PID controllers to generate body torques.

Position control follows a feedback + feedforward structure:

acmd = Kp(xdes − x) + Kd(vdes − v) + ades

Desired accelerations are mapped to attitude and thrust commands using quadrotor dynamics:

ax → θdes (pitch)

ay → φdes (roll)

az → thrust

This structure enables the controller to track dynamically consistent trajectories rather than static setpoints.

Trajectory generation & waypoint navigation

Extended the system from point-to-point control to trajectory-based autonomous navigation.

  • Implemented quintic polynomial trajectory generation between waypoints.
  • Ensured continuity in position, velocity, and acceleration.
  • Generated time-parameterized trajectories x(t), v(t), a(t).
  • Designed a waypoint manager that advances when proximity thresholds are met and resets trajectory timing for each segment.
  • Introduced time-scaling (T selection) based on distance, velocity, and acceleration constraints to ensure smooth, physically feasible motion.

This approach eliminates discontinuities in acceleration and significantly improves tracking performance, stability, and realism compared to direct waypoint commands.

PID control & tuning

  • Implemented reusable PID controllers for all translational and rotational axes.

Tuned gains to achieve:

  • fast recovery from large disturbances
  • minimal oscillation in attitude
  • stable trajectory tracking under wind disturbances

Debugged and resolved:

  • acceleration spikes from discontinuous references
  • oscillatory coupling between translation and rotation
  • instability from overly aggressive trajectory timing

Software architecture

Refactored the project into a modular structure inspired by real UAV flight stacks: 

6DOF_Drone_Sim/
├── drone_main.py                       # simulation loop, logging, plotting
├── dynamics_fun.py                     # 6-DOF rigid body dynamics (quaternion)
├── integrator_fun.py                   # numerical state integrator
├── disturbances_fun.py                 # wind / gust disturbance model
├── controllers/
│   ├── outerLoop.py                    # position control + waypoint manager
│   ├── innerLoop.py                    # attitude control (roll, pitch, yaw)
│   ├── trajectoryGeneration_fun.py     # quintic polynomial trajectory generator
│   └── PID.py                          # reusable PID class with anti-windup
└── transform/
    ├── quat2euler_fun.py               # quaternion → Euler angles
    ├── euler2quat_fun.py               # Euler angles → quaternion
    ├── body2global_fun.py              # body → world frame rotation
    ├── angularAccel_fun.py             # rotational dynamics (Euler’s equation)
    └── Tcalc_fun.py                    # mixer: total thrust + torques → rotor thrusts
  • Separated trajectory generation, control, and dynamics into independent modules.
  • Structured interfaces as (state, reference, parameters).
  • Maintained persistent controller and trajectory state across simulation steps.

This design improves scalability and reflects real-world systems such as PX4 Autopilot and ArduPilot.

Integration & testing

Validated system performance through simulation:

  • Stabilization from large initial disturbances (e.g., 45° roll/pitch).
  • Smooth waypoint traversal using quintic trajectories.
  • Tracking performance under wind disturbances.
  • Comparison of desired vs. actual position, velocity, and acceleration.

Used data visualization (Matplotlib) to analyze:

  • trajectory tracking accuracy
  • control effort and actuator behavior
  • transient response and stability margins

Results & lessons learned

  • Trajectory generation is essential for stable, realistic quadrotor motion—waypoints alone are insufficient.
  • Feedforward acceleration dramatically reduces controller workload and improves tracking.
  • Time-scaling (trajectory duration) directly impacts system stability and physical feasibility.
  • Cascaded control architectures provide modularity and robustness.
  • Coupling between translational and rotational dynamics must be carefully managed.
  • Clean software structure becomes critical as autonomy features are added.
