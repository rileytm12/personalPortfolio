Overview

This project focuses on coordinating a network of three differential-drive mobile rovers to autonomously complete waypoint-based missions in a shared environment. Each rover must follow planned paths while maintaining stability, accuracy, and synchronization with the rest of the system.

The system integrates onboard sensing, state estimation, and real-time control to ensure accurate localization and trajectory tracking. Success is measured by minimizing path tracking error, maintaining stable wheel velocity control, and achieving consistent multi-rover coordination in both simulation and hardware testing.

My work on the project

Path tracking algorithm development:

Modeled and compared both Pure Pursuit and Stanley controllers in MATLAB to evaluate tracking performance, stability, and suitability for our rover platform. Based on this analysis, selected Pure Pursuit as the primary approach.

C++ implementation of adaptive Pure Pursuit:

Developed a variable lookahead Pure Pursuit controller in C++ that dynamically adjusts lookahead distance based on path curvature. This improves tracking accuracy on sharp turns while maintaining stability at higher speeds. The controller outputs real-time linear and angular velocity commands for the rover.

Controls & estimation integration:

Contributed to the implementation of a fuzzy PID controller for wheel speed regulation and assisted in integrating a Kalman filter for improved state estimation and localization.

Hardware integration:

Assisted in integrating the software stack with rover hardware, including deployment on an embedded system using the NVIDIA Jetson Orin Nano and communication with a microcontroller (STM32 microcontroller) for low-level motor control.

Software & ROS2 architecture

The system is built on a modular ROS2 architecture spanning high-level mission planning to low-level actuation:

ARES GNC layout
  • High-level nodes generate mission objectives and waypoint sequences
  • Planning nodes define feasible paths for each rover
  • The path tracking controller (C++) computes velocity commands
  • Estimation nodes (Kalman filter) fuse sensor data for pose updates
  • Low-level nodes interface with motor drivers via embedded hardware

Path tracking & control

Controller selection:

Evaluated Pure Pursuit and Stanley methods in MATLAB simulations. Pure Pursuit was selected for its robustness and smoother behavior under our system constraints.

Adaptive Pure Pursuit:

Implemented a curvature-based variable lookahead distance: larger lookahead for straight paths (improved stability) and smaller lookahead for high curvature (improved accuracy).

Kinematics:

Applied differential-drive kinematics to convert path geometry into linear and angular velocity commands, while respecting actuator constraints (e.g., max angular velocity and acceleration limits).

Fuzzy PID control:

Designed to regulate wheel velocities under varying terrain and load conditions, using rule-based gain adjustments to improve responsiveness compared to fixed-gain PID.

Kalman filter:

Integrated to improve localization by fusing sensor inputs (e.g., wheel odometry and onboard sensors), reducing drift and noise in pose estimation.

Integration & testing

Conducted controller validation in MATLAB simulations before deployment. Tested the full system in ROS2-based simulation and on physical rover hardware.

Assisted in debugging integration issues including:

  • Timing inconsistencies between nodes
  • Communication between embedded systems and ROS2
  • Hardware/software interfacing between Jetson and STM32

Results & lessons learned

  • Adaptive Pure Pursuit improved tracking performance on curved paths compared to fixed-lookahead implementations.
  • Integration of estimation and control significantly reduced localization error and improved trajectory stability.
  • Learned the importance of tuning control algorithms in both simulation and real-world conditions, as hardware constraints introduced non-ideal behaviors not captured in initial models.
  • Gained experience working across software and hardware boundaries in a multi-team robotics system.
